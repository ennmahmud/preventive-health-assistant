# push_changes.sh

This script helps you safely prepare and push the current workspace to a remote Git repository. It runs on macOS with zsh (the default shell in this workspace). It is interactive and conservative by default — it will NOT force-push unless you explicitly choose that option and confirm.

Usage:

```zsh
cd /path/to/preventive-health-assistant
./scripts/push_changes.sh
```

What it does:
- Optionally creates a timestamped zip backup of the workspace
- Verifies or initializes a Git repository
- Shows the current branch, remotes, recent commits, and working tree status
- Lets you switch or create a branch
- Stages all changes and commits with an entered or autogenerated message
- Lets you add or update a remote (default: origin)
- Fetches from the remote and shows divergence if present
- Attempts a safe push (normal push). If push fails due to divergence, it offers options to pull/merge, pull/rebase, or force-with-lease (requires explicit confirmation)
- Shows final verification commands

Notes:
- The script stages all changes with `git add -A`. If you prefer to selectively stage, do that manually and re-run the script.
- For authentication, prefer SSH. To test SSH: `ssh -T git@github.com`.
- If your repository contains large files (big binary files), consider using Git LFS.

Safety:
- The script creates a backup by default. It avoids unconditional force pushes.
- When force-push is selected, it requires typing the exact phrase `I understand` to proceed.
